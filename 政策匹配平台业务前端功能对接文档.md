## 1. 文档依据

本文档按以下材料逐项核对生成：

本文档只描述当前代码和接口已经能够支撑的功能。原型中存在但后端尚未形成闭环的交互，不作为本期前端交付内容。

## 2. 功能状态定义

| 状态 | 含义 | 前端处理 |
|---|---|---|

## 3. 当前业务范围

本期前端实现三个页面：

1. 园区政策服务首页。
2. 政策详情/政策找企业页面。
3. 企业详情/企业找政策页面。

三个页面均可接入右侧 AI 政策申报顾问，支持：

- 预设问题。
- 手动输入。
- 流式回答。
- 页面对象上下文传递。
- 清空并新建会话。
- 历史消息恢复。

本期不实现：

- 政策规则修改、确认和发布。
- 匹配重算任务。
- 企业培育计划编辑。
- AI 生成并保存服务任务。
- 短信、微信、站内信提醒。
- 材料模板预填。
- 企业跟踪、待办任务和任务状态管理。
- 匹配报告、表格或材料清单导出。

## 4. 前端调用边界

```text
页面业务数据
  → information-brief Java REST API

AI 对话
  → Vela Copilot Session API
  → Vela AG-UI
  → MCP
  → information-brief Java REST API
```

前端不得直接调用 MCP，也不得使用页面已有分数、条件状态或企业指标替代 AI 的权威查询。

### 4.1 本地地址

| 服务 | 地址 | 前端用途 |
|---|---|---|
| Java | `http://127.0.0.1:31454` | 页面列表、详情和匹配数据 |
| Vela Biz | `http://127.0.0.1:8000/api/v1` | AI Session、AG-UI 和历史消息 |

### 4.2 当前鉴权

- Java `/api/policy-copilot/v1/**`：当前前端联调阶段免鉴权。
- Vela `/api/v1/**`：必须携带 `Authorization: Bearer <vela-jwt>`。
- 浏览器不携带 MCP 内部密钥或 Java 服务 Token。

### 4.3 Java 响应判断

Java 使用统一响应：

```ts
interface JavaResponse<T> {
  code: number;
  msg: string;
  data: T;
}
```

前端必须同时检查 HTTP 状态和 `body.code`。HTTP 200 不代表业务一定成功。

### 4.4 内部标识使用规则

以下字段只作为路由、查询和上下文参数使用，不得作为业务文案展示：

- `enterpriseId/entUid`
- `policyId`
- `projectId`
- `matchId`
- `indicatorId`
- `fieldName`

页面必须展示企业名称、政策名称、项目名称和条件名称。

### 6.3 政策详情接口

```http
GET /api/policy-copilot/v1/policies/{policyId}/context
```

推荐前端模型：

```ts
interface PolicyContext {
  title: string;
  titleFull?: string;
  documentNo?: string;
  issuingOrg?: string;
  region?: string;
  level?: string;
  policyAttribute?: string;
  deadline?: string;
  materials?: string;
  process?: string;
  projects: Array<{
    projectId: number;
    projectName: string;
    subjectType?: string;
    groupsRelation?: string;
    conditions: Array<{
      indicator: string;
      compare?: string;
      fieldVal?: string;
      conditionRole?: 'must' | 'optional' | string;
      sourceClause?: string;
      matchStatus?: string;
    }>;
  }>;
}
```

条件表字段映射：

| 原型列 | 接口字段 |
|---|---|
| 指标名称 | `indicator` |
| 指标值/条件内容 | `compare + fieldVal` |
| 必要指标 | `conditionRole === 'must'` |
| 政策原文依据 | `sourceClause` |

不要展示 `fieldName/indicatorId/indicatorScore`。

### 6.4 政策找企业接口

```http
POST /api/policy-copilot/v1/matches/enterprises-by-policy
Content-Type: application/json

{
  "policyId": 4,
  "projectId": null,
  "keyword": null,
  "matchLevels": ["high"],
  "pageNum": 1,
  "pageSize": 20
}
```

切换项目后传对应 `projectId`。切换高中低标签时分别传：

```ts
high   -> ['high']
medium -> ['medium']
low    -> ['low']
all    -> ['high', 'medium', 'low']
```

空结果时展示接口 `message`：

```text
暂无最新匹配结果，请先执行企业找政策或后台预计算
```

不得自行扫描企业库或展示原型静态企业名单。

### 6.5 企业匹配详情

```http
GET /api/policy-copilot/v1/matches/{matchId}/evidence
GET /api/policy-copilot/v1/matches/{matchId}/gaps
```

详情按以下四组展示：

- 已满足：`evidenceStatus=satisfied`
- 明确不满足：`evidenceStatus=unsatisfied`
- 待补充或待确认：`evidenceStatus=unknown`
- 不适用：`evidenceStatus=not_applicable`

当前 `gaps.materials[]` 是通用提示，不是正式材料清单。页面标题应使用“材料准备提示”，不能标成“缺失材料明细”。

### 6.6 政策页面 AI 上下文

```json
{
  "scene": "policy_detail",
  "policyId": "4",
  "projectId": "可选项目标识"
}
```

推荐预设问题：

- 我想了解这项政策。
- 这项政策有哪些申报项目和必要条件？
- 当前哪些企业建议优先查看？
- 为什么某家企业属于高/中/低匹配？

当用户只问“了解政策”时，AI 只查询政策详情，不继续查询匹配证据。

### 7.4 企业找政策接口

```http
POST /api/policy-copilot/v1/matches/policies-by-enterprise
Content-Type: application/json

{
  "enterpriseId": "opaque-enterprise-key"
}
```

返回的每一条结果是“企业 + 政策 + 项目”匹配结果。政策相同但项目不同必须保留为不同列表行。

列表字段映射：

| 原型列 | 当前字段/处理 |
|---|---|
| 政策名称 | `policyTitle` |
| 项目名称 | `projectName`，新增为必要展示 |
| 政策类型 | 当前缺失，本期不展示 |
| 匹配度 | `score` |
| 匹配等级 | `matchLevel` |
| 匹配情况 | 四态计数 + 按需加载 evidence |
| 主要差距 | 点击详情后加载 gaps，不建议列表逐条请求 |

当前请求 DTO 中虽存在 `matchLevels/policyAttribute/sort/pageNum/pageSize`，实时匹配流程尚未实际应用这些参数，前端不得依赖它们进行服务端筛选和分页。

## 8. AI 助手统一对接

### 8.1 创建 Session

进入三个业务页面时，为当前业务对象创建或复用一个 Policy Copilot Session：

```http
POST /api/v1/copilots/policy-matching/sessions
Authorization: Bearer <vela-jwt>
Content-Type: application/json

{
  "title": "政策匹配助手"
}
```

响应：

```ts
interface CopilotSession {
  session_id: string;
  profile: 'policy-matching';
  profile_version: string;
  runtime_url: '/api/v1/agui/run';
}
```

### 8.2 AG-UI 请求

推荐使用 CopilotKit/AG-UI 客户端。等价请求：

```json
{
  "threadId": "sess_xxx",
  "runId": "前端UUID",
  "state": {},
  "messages": [
    {
      "id": "msg_前端UUID",
      "role": "user",
      "content": "这家企业可以优先关注哪些政策？"
    }
  ],
  "tools": [],
  "context": [
    {
      "description": "policy-copilot-context",
      "value": "{\"scene\":\"enterprise_detail\",\"enterpriseId\":\"企业标识\"}"
    }
  ],
  "forwardedProps": {}
}
```

请求地址：

```text
POST /api/v1/agui/run
Accept: text/event-stream
```

### 8.3 页面上下文规则

允许字段：

```ts
interface PolicyCopilotPageContext {
  scene?: 'enterprise_detail' | 'policy_detail' | 'park_dashboard' | 'match_result';
  enterpriseId?: string;
  policyId?: string;
  projectId?: string;
  matchId?: string;
  regionId?: string;
  parkId?: string;
}
```

- `value` 必须是 JSON 对象字符串。
- 长度不得超过 4096 字符。
- 不允许增加未定义字段。
- 不传企业指标、政策条件、分数、等级、材料状态或用户 Token。

### 8.4 匹配详情上下文

用户在某条匹配结果点击“AI 解释”时传：

```json
{
  "scene": "match_result",
  "enterpriseId": "企业标识",
  "policyId": "政策标识",
  "projectId": "项目标识",
  "matchId": "匹配标识"
}
```

页面仍不展示这些内部标识。

### 8.5 清空会话

原型“清空”按钮本期建议：

1. 清除当前页面内存中的聊天展示。
2. 重新调用 Copilot Session API 创建新 Session。
3. 后续 AG-UI 使用新的 `threadId`。

不要只清空 DOM 后继续使用旧 Session，否则历史上下文仍会参与回答。

### 8.6 历史消息

```http
GET /api/v1/sessions/{sessionId}/messages
Authorization: Bearer <vela-jwt>
```

Policy Profile 已隐藏工具轨迹。前端只展示 `user/assistant` 消息和最终业务回答，不渲染工具名、工具参数、原始 MCP 结果或异常栈。

## 9. 匹配结果统一展示规则

### 9.1 匹配等级

| 服务端值 | 页面文案 | 分数范围 |
|---|---|---|
| `high` | 高匹配 | `score >= 80` |
| `medium` | 中匹配 | `50 <= score < 80` |
| `low` | 低匹配 | `20 <= score < 50` |
| `none` | 不匹配 | `score < 20` |

前端直接使用服务端 `score/matchLevel`，不重新计算等级。

### 9.2 准入状态

| 服务端值 | 页面文案 |
|---|---|
| `eligible` | 具备申报潜力 |
| `need_confirmation` | 必要条件待确认 |
| `not_eligible` | 必要条件明确不满足 |

“高匹配”不等于“保证可以申报”。页面不能使用“可直接申报”“一定通过”等绝对表述。

### 9.3 四态证据

| 服务端值 | 页面分组 |
|---|---|
| `satisfied` | 已满足 |
| `unsatisfied` | 明确不满足 |
| `unknown` | 待补充或待确认 |
| `not_applicable` | 不适用 |

禁止把 `unknown` 归入“不满足”。缺失材料也不能直接解释为企业不符合。

## 10. 搜索和对象选择

### 10.1 搜索企业

```http
GET /api/policy-copilot/v1/enterprises/search?query=企业名称或信用代码
```

多候选时展示企业名称和信用代码，让用户选择。

### 10.2 搜索政策

```http
GET /api/policy-copilot/v1/policies/search?query=政策名称或简称
```

接口支持中英文引号和常见标点差异。

### 10.3 搜索项目

```http
GET /api/policy-copilot/v1/projects/search?policyQuery=政策名称&query=项目名称
```

### 10.4 解析匹配对象

```http
GET /api/policy-copilot/v1/matches/resolve
```

可传 `query/enterpriseQuery/policyQuery/projectQuery`。多条结果时按企业、政策和项目名称让用户选择，不要求用户输入内部标识。

## 11. 错误和空状态

### 11.1 Java

| 场景 | 前端文案 |
|---|---|
| 搜索无结果 | 未找到匹配对象，请尝试完整名称或信用代码 |
| 政策找企业为空 | 当前暂无最新企业匹配结果，需要先完成企业画像分析或匹配计算 |
| 企业找政策为空 | 当前暂无匹配结果，请确认企业画像数据是否已准备 |
| `MATCH_NOT_FOUND` | 当前没有查到有效匹配结果 |
| HTTP 5xx/网络失败 | 业务数据暂时无法获取，请稍后重试 |

不要向用户展示 Java 类名、接口路径、内部错误码和异常堆栈。

### 11.2 Vela AI

| 场景 | 前端处理 |
|---|---|
| Session 创建失败 | 禁用 AI 输入，显示“AI 助手暂不可用” |
| `RUN_ERROR` | 结束 loading，展示业务化错误文案 |
| AG-UI 中断 | 根据 response schema 展示选择或确认控件 |
| 页面上下文非法 | 记录前端错误，提示重新进入页面 |

## 13. 本期前端验收标准

### 首页

- 能展示真实覆盖政策数和当前重点政策。
- 占位统计不显示为真实业务成果。
- 无快讯时展示空状态。
- AI 可围绕当前政策库概览进行流式回答。

### 政策找企业

- 能展示政策详情和分项目条件。
- 能按项目、高中低等级和企业关键词查询企业。
- 能查看一条匹配结果的四态证据和差距。
- 无预计算结果时不展示静态企业名单。
- 不出现任务、提醒、预填和导出假交互。

### 企业找政策

- 能展示企业基本画像、来源、更新时间和缺失项。
- 能展示政策 + 项目级匹配结果。
- 能按匹配等级进行前端筛选。
- 能打开匹配证据和差距详情。
- 不展示模拟培育计划和收益。

### AI 助手

- 三个页面都可创建独立 Policy Copilot Session。
- 预设问题和手动输入均可流式输出。
- 页面上下文能正确定位企业、政策、项目和匹配结果。
- 清空后使用新 Session。
- 回答不出现 MCP 工具名、内部标识、数据库字段或原始异常。
- `unknown` 始终解释为待补充或待确认。
